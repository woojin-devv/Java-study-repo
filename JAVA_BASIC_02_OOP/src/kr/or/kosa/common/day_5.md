# Java 프로그래밍 및 객체지향 요약

## Static 및 Instance 자원
- Static 자원은 일반 함수에서 접근이 가능하다.

## 데이터 전달 방식
- Call by Value와 Call by Reference의 개념을 구분했다.

## Method Overloading
- 동일한 이름의 메서드를 매개변수의 타입이나 개수에 따라 다르게 정의하는 방식을 학습했다.

## 객체지향 프로그래밍 (OOP)
- 여러 개의 작은 설계도(객체)를 모아서 하나의 큰 시스템을 구축하는 방식이다.
- 추상화, 다형성, 상속, 캡슐화는 OOP 그 자체가 아니라 객체지향의 주요 특징에 해당한다.

## 객체지향의 특징
- 상속: 코드의 재사용성을 높이는 특성이다.
- 캡슐화: 데이터를 외부로부터 보호하며 private 제어자와 getter, setter 메서드를 통해 데이터를 간접적으로 관리한다.
- 다형성
	- 상속 ( 상속 관계에서 부모타입 )

## Method Overloading
- 기능이 유사한 함수들을 출력 타입이나 파라미터에 따라 여러 개 생성할 경우 함수 이름이 복잡해지는 문제가 발생했다.
- 이를 해결하기 위해 함수 이름을 하나로 통일하고, 입력받는 파라미터의 타입이나 개수에 따라 적절한 함수가 호출되도록 구성했다.
- 이러한 방식을 Method Overloading이라 정의했다.
- 예시 : System.out.println();
