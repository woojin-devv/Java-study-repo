## 1. Java의 Method 유형

> 메서드는 입력(parameter)과 출력(Return)의 유무에 따라 4가지 조합으로 나뉜다.

| 유형                    | 형태 (예시)      | 특징                                                                      |
| :---------------------- | :--------------- | :------------------------------------------------------------------------ |
| **1. 입력(X), 출력(X)** | `void m()`       | **단순 실행형.** 로그 출력이나 상태 변경에 사용.                          |
| **2. 입력(O), 출력(X)** | `void m2(int i)` | **데이터 수신형.** 외부 데이터를 받아 내부 처리에 사용.                   |
| **3. 입력(X), 출력(O)** | `int m3()`       | **데이터 조회형.** 고정된 값이나 객체의 상태를 조회(Getter 등)할 때 사용. |
| **4. 입력(O), 출력(O)** | `int m4(int d)`  | **데이터 가공형.** 가장 전형적인 함수. 데이터를 가공하여 결과를 돌려줌.   |

### Method

1. 단일 책임 : 하나의 메서드 당 하나의 기능만..
2. 호출의 원리 : 메서드는 반드시 누군가 불러줘야 메모리에 올라가고 작동함
3. 유연한 리턴 : 객체 리턴 가능 (이걸 **참조형**이라고 함)

## 2. 접근 제한자 (Public Vs. Private)

- Public : 외부 객체에서 호출 가능함.
- Private : 클래스 내부에서만 사용되는 로직
  - 캡슐화를 실현하는 핵심 도구이다.

## 3. 리턴 (Return)의 다양한 형태

> 자바에서 리턴 타입은 단순히 숫자에 그치지 않음

- 기본형 : int, boolean
- 참조형 : Car, int[], List() 등
- 인터페이스 : List, Map 등 (유연한 설계를 위해 권장됨)

### 좋은 메서드를 짜기 위한 습관 ..

> 요구 사항 : a와 b를 받아서 max 값을 반환하는 메서드를 짜려고 한다.

- 기존 코드

```java

public int maxNum(int a, int b) {
    if (a > b) {
        return a; // 첫 번째 리턴
    } else {
        return b; // 두 번째 리턴
    }
}

```

- 리턴은 되도록 많이 안하는 것이 좋음
